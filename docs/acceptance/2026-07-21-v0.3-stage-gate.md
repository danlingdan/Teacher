# v0.3.0 阶段门禁验收

## 1. 验收结论

`v0.3.0` 题库、隔离练习、确定性评测、分级提示和基础学习进度已完成，阶段门禁通过。

本次按单人开发流程直接在 `main` 开发，每个完整功能单独提交，不创建 PR。开发期间按约定只做编译检查；全部功能完成后集中执行测试与桌面走查。

## 2. 已交付能力

- `app.db` schema 版本 3：题目数据集、题目、练习会话和尝试记录。
- 教师题库：创建、编辑、乐观版本保护、复制默认停用、启停、版本 1 JSON 题包原子导入导出。
- 20 道幂等内置题：基础查询、筛选、聚合、连接、子查询各 4 道。
- 学生闯关：选题、隔离 SQLite 数据集、运行、提交、重置和最多三级确定性提示。
- SQL 安全：学生 SQL 仅允许单条 `SELECT`，启用 `query_only`、10 秒超时和结果行上限。
- 确定性评测：列、行、值、重复行、顺序、指定行数及必需 SQL 结构；等价写法按结果语义通过。
- 隐私边界：学生 DTO 和反馈不包含参考 SQL、初始化 SQL、参考结果或正确值。
- 尝试记录：SQL、运行/提交状态、结果摘要、反馈、耗时、错误码和提示次数。
- 基础学习进度：会话、尝试、提交通过率、平均提交耗时、提示、完成题目和逐题进度。
- 生命周期恢复：正常退出关闭活跃会话；蓝屏或强退后的残留会在下次启动时收尾并清理。

## 4. 真实 JavaFX 走查

通过 `mvn -q javafx:run` 启动实际桌面应用并完成以下路径：

1. 首页显示新增导航，窗口在 1180×720 正常布局。
2. “闯关练习”加载 20 道题及学生可见说明。
3. 开始题目后创建隔离数据集；输入 `select name from student order by id` 并运行，动态结果表显示 `name` 列和学生数据。
4. “教师题库”加载 20 道题、数据集和完整编辑表单；修正样式类后表单保持统一深色主题。
5. “学习进度”读取刚才产生的 1 个会话和 1 次运行尝试，通过率分母未错误包含仅运行记录。
6. 应用正常关闭；中断恢复修复后重新初始化确认 `app-data/exercise-sessions` 残留文件数为 0。

未在走查中修改、停用或导入正式题目；运行数据仅位于被 Git 忽略的 `app-data/`。

## 5. 安全与仓库检查

- `git ls-files app-data target` 无输出，运行数据库和构建产物未被跟踪。
- 本地 MySQL 密码精确扫描无匹配，未写入代码、文档或测试。
- 题包导入使用预编译参数和单事务；冲突或依赖错误会整体回滚。
- 会话清理器只删除会话目录中符合 UUID 命名规则的 `.db`、`.db-wal` 和 `.db-shm` 文件。
- 所有学生 SQL 仍经过 Java 风险分析，AI 不接触 JDBC Connection，也不能直接执行 SQL。
