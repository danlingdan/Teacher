# SQLTeacher v1.1 迭代计划表

> 制定日期：2026-07-22
> 建议周期：2026-07-23 至 2026-09-16（8 周）
> 起始基线：`v1.0.0`（228 项自动化测试通过）
> 版本定位：协作教学与可选云端 AI 增强版

## 1. 迭代目标

在不改变 `v1.0.0` SQL 安全边界的前提下，增加多教师、多班级、登录与权限、在线同步，以及用户自带 API 的可选网络 AI 能力。

本版本完成后，教师可组织班级、分配任务并查看本班数据；学生可通过账号在设备间同步学习进度；用户可自行配置受支持网络 AI 的 API，在本地预览安全检查后的 SQL 草案或解释。

## 4. 功能验收表

| 功能 | 正常场景 | 失败/边界场景 | 安全要求 |
|---|---|---|---|
| 登录与权限 | 用户可登录并根据管理员、教师、学生角色使用页面与 API。 | 无效密码、过期会话、禁用用户与越权请求均有统一响应。 | 密码使用现代单向哈希；令牌短期有效且可撤销；拒绝默认访问。 |
| 班级协作 | 多名教师可管理同一班级，学生仅加入指定班级。 | 重复邀请、已移除成员和跨班级访问得到明确反馈。 | 所有班级、任务、统计和学习记录均执行服务端成员关系校验。 |
| 在线同步 | 多端可同步任务、尝试和进度。 | 离线、重复请求、并发修改和冲突可恢复且可说明。 | HTTPS 传输；服务端按用户和班级隔离数据；同步不传数据库密码或 API Key。 |
| 网络 AI | 用户填入 API Key 后可获得 SQL 草案或教学解释。 | 无效 Key、超时、限流、Provider 故障均可降级。 | Key 仅保留本地安全存储；敏感数据最小化发送；模型输出必须经本地安全链路。 |
| 题库与反馈 | 合法题包可预检/导入；学生得到可执行的非答案反馈。 | 题包损坏、字段缺失、SQL 错误和会话过期均可定位。 | 失败导入回滚；不暴露标准答案、隐藏数据或连接信息。 |
| UI 可用性 | 关键任务的主操作、状态和下一步在常用分辨率下清晰可见。 | 空数据、加载失败、长文本、窄窗口和键盘操作均有可理解反馈。 | 不用仅颜色传达风险或错误；敏感字段默认掩码。 |

## 5. 云服务与账号体系部署计划

### 5.1 部署目标与访问边界

| 项目 | 计划 |
|---|---|
| 部署服务器 | `8.130.47.235`，仅作为 v1.1 云端 API、账号体系与同步服务的部署目标。 |
| 对外地址 | 优先绑定正式域名并通过 HTTPS 访问；IP 仅用于首次 SSH 运维、DNS 生效前验证或受限测试。客户端不应长期硬编码 IP。 |
| 网络入口 | 防火墙/安全组仅开放 `80/tcp`、`443/tcp` 和受来源限制的 `22/tcp`；应用端口和数据库端口不直接暴露公网。 |
| 服务端职责 | 认证、角色授权、班级成员关系、任务分配、同步 API、审计日志和服务端数据迁移。 |
| 不上云的数据 | 用户本地数据库连接密码、用户配置的网络 AI API Key、未获授权上传的课程原文和本地数据库文件。 |

### 5.2 服务端配置步骤

| 顺序 | 配置项 | 服务器端执行内容 | 验收标准 |
|---|---|---|---|
| 1 | 操作系统与账户 | 创建专用非 root 部署账户；禁用 SSH 密码登录，使用密钥登录；安装安全更新和时间同步。 | root 无法直接远程登录；部署账户具备最小权限。 |
| 2 | 网络与 TLS | 在 `8.130.47.235` 配置安全组；将域名 A 记录指向该 IP；使用 Nginx/Caddy 反向代理和受信任证书启用 HTTPS。 | HTTP 自动跳转 HTTPS；TLS 证书有效；仅 80/443 对公网开放。 |
| 3 | 运行环境 | 以容器或受控系统服务运行 API；配置健康检查、自动重启、结构化日志和资源限制。 | 服务重启后自动恢复；健康检查端点仅返回非敏感状态。 |
| 4 | 服务端数据库 | 使用独立数据库账号和独立数据卷；仅允许本机/私网访问；启用自动备份、加密备份和恢复演练。 | 公网不能连接数据库；从备份可恢复到隔离环境。 |
| 5 | 账号体系 | 创建 `users`、`roles`、`user_roles`、`classes`、`class_teachers`、`class_students`、`refresh_tokens` 和审计表；密码使用 Argon2id 或 bcrypt 哈希。 | 注册、登录、登出、禁用用户、刷新令牌和角色变更均有集成测试。 |
| 6 | 授权与 API | 所有班级、任务、尝试和统计接口在服务端校验身份、角色及成员关系；令牌设置过期、刷新和撤销机制。 | 跨用户、跨角色、跨班级请求返回拒绝，且不会泄露资源是否存在。 |
| 7 | 发布与密钥 | 使用服务器环境变量或受控密钥服务保存 JWT 签名密钥、数据库凭据等服务端密钥；通过 CI/CD 或受控部署脚本发布。 | 仓库、日志、镜像和桌面客户端均不含服务端密钥。 |
| 8 | 监控与应急 | 配置访问日志脱敏、错误告警、备份失败告警、限流和审计保留期限；准备密钥轮换及账号冻结流程。 | 可追踪异常登录与同步失败；演练令牌撤销和密钥轮换。 |

### 5.3 账号与权限最小模型

| 角色 | 允许操作 | 不允许操作 |
|---|---|---|
| 管理员 | 管理用户状态、全局角色、系统审计与服务配置。 | 直接读取用户密码、API Key 或本地数据库连接信息。 |
| 教师 | 创建/管理自己参与的班级、维护本班任务、查看本班学情。 | 管理非所属班级；访问其他班级学生数据。 |
| 学生 | 加入获邀班级、领取任务、提交尝试、查看自己的进度。 | 修改题库、分配任务、读取其他学生明细。 |

### 5.4 客户端与服务器同步规则

- 桌面端只保存短期访问令牌和可撤销的刷新令牌；令牌使用操作系统安全存储，不能写入普通配置、备份或日志。
- 用户的网络 AI API Key 只保留在该用户本机安全存储中；云端同步协议不定义该字段。
- 学习尝试采用稳定 ID、创建时间、修改版本和幂等请求键；服务端返回冲突状态及可合并字段，客户端保留待同步队列并允许用户重试。
- 数据传输仅使用 HTTPS；服务端对登录、刷新令牌和同步接口实施速率限制、请求大小限制和审计。
- 首次部署前需由项目负责人确定域名、云服务器操作系统、服务端技术栈、数据库类型、备份保留期和学生数据合规要求；这些选择不得由客户端代码隐式决定。

## 6. 执行规则

- 每周只推进一个主要目标；未关闭的 P0 问题优先于新功能。
- 每个功能按“契约/数据 → 基础设施 → 桌面端 → 测试与文档”完成纵向闭环。
- SQL、AI、认证、授权、同步、密钥存储、云端部署和数据迁移相关改动必须运行相应聚焦测试；阶段收敛时运行 `mvn test`。
- 不记录或提交真实密码、访问令牌、API Key、学生数据、运行期数据库和部署产物。
- 任何新数据结构变更必须提供 schema migration、旧库升级测试和恢复策略。
- 网络 AI 只使用用户主动配置的 API Key；密钥不得上传到本项目服务端、同步到其他设备、写入日志、备份或导出。
- AI 仅可生成草案或解释，不能参与最终判题、直接执行 SQL 或取得 JDBC 连接。

## 7. 发布条件

- `mvn test` 全量通过，且新增 P0 测试覆盖越权拒绝、跨班级隔离、同步幂等/冲突恢复、题包失败回滚、SQL 安全和网络 AI 降级。
- 登录、角色授权、多教师班级、任务分配和进度同步均具备端到端验收记录。
- API Key 不会被云端服务接收、同步、备份、导出或记录到日志；网络 AI 故障时应用安全降级。
- `8.130.47.235` 上的服务已通过裸 IP HTTP 测试入口、最小应用端口暴露、服务端数据库非公网暴露和越权拒绝验证；按当前测试优先要求暂缓域名与 HTTPS，正式教学前必须补齐 HTTPS 和备份恢复演练。
- UI 走查覆盖主导航、登录、班级、任务、同步、网络 AI、空/加载/错误状态、键盘路径、窄窗口和中文长文本。
- SQLite 核心练习、确定性评测、统计与本地知识检索在云端服务、网络 AI、Ollama 或外部数据库不可用时仍可使用。
- `v1.1.0` 发布说明、验收报告、用户/教师手册和已知限制与实际实现一致。

## 8. 风险与裁剪顺序

| 风险 | 应对 | 工期不足时的裁剪 |
|---|---|---|
| 账号或班级数据越权 | 以服务端授权为唯一边界，建立跨角色与跨班级的拒绝测试。 | 不裁剪授权测试；裁剪页面美化。 |
| 同步冲突或网络不稳定 | 采用版本号、幂等请求和本地待同步队列，提供重试与状态提示。 | 保留学习记录同步，裁剪实时刷新。 |
| API Key 或敏感数据泄露 | 本地安全存储、字段白名单、日志脱敏和发送内容最小化。 | 不裁剪密钥保护；裁剪 Provider 数量。 |
| 云服务器被直接攻击或误配置 | 最小端口、HTTPS、非 root 部署、密钥登录、备份、限流与告警；应用/数据库不直接暴露。 | 不裁剪基础加固；裁剪非必要运维面板。 |
| UI 改造影响主流程 | 先以关键任务走查和现有 FXML/控制器测试固化行为，再做小批量视觉调整。 | 保留状态与可用性，裁剪动画和装饰性效果。 |
| 题包兼容影响既有题库 | 先建立旧题包夹具和导入回归。 | 保留兼容与错误定位，裁剪筛选界面美化。 |
| 反馈文案泄露答案 | 使用规则化错误码和脱敏样例评审。 | 保留安全反馈，裁剪 AI 辅助解释。 |
| 发布收敛被新需求打断 | 新需求进入 v1.2 候选池。 | 优先完成 P0 验证、测试和文档。 |
