> 制定日期：2026-07-22
>

>

> 版本定位：正式教学可用与稳定性增强版

## 1. 迭代目标

在保留 `v1.1.0` 本地优先、SQL 安全与可选云端 AI 边界的前提下，将当前联调性质的云端协作能力提升为可在真实教学场景中受控使用的服务。

本版本完成后，学校可通过受 HTTPS 保护的正式服务组织班级教学；学生可在应用重启后安全恢复登录并可靠同步学习记录；教师可按任务周期查看班级完成情况并导出教学记录；发生网络、服务或本地数据问题时，系统具备明确提示、可恢复路径和可追溯运维记录。

## 3. 关键设计原则

- **本地优先不变**：云端不可用时，SQLite 练习、确定性评测、本地知识检索和本地事件记录仍可用；同步仅进入待处理状态，不能阻断学习。
- **服务端仍是授权边界**：桌面端隐藏按钮或页面不是安全控制；班级、任务、尝试、统计和导出必须在服务端验证身份、角色和成员关系。
- **令牌与密钥分离**：会话令牌可在操作系统安全存储中持久化；网络 AI API Key 保持仅本机、仅用户主动提供，不能进入同步、备份、导出、日志或服务端请求体。
- **任务状态以服务端时间为准**：发布、截止、撤回和归档由服务端统一判定，客户端只缓存展示；离线提交保留本地时间和幂等键，恢复后由服务端返回最终状态。
- **导出最小化**：教师只可导出所管理班级的必要教学字段；CSV 不含密码、令牌、API Key、原始 SQL 中的敏感连接信息或其他班级学生明细。
- **变更可恢复**：服务端和本地 schema 变化均应提供可重复 migration、升级测试、备份及恢复步骤；恢复演练不使用真实学生数据。
- **SQL/AI 安全红线不变**：多语句和禁止级 SQL 继续拦截，高风险 SQL 继续要求明确确认；AI 输出只能作为经本地校验后的草案或解释，绝不直接访问 JDBC 或执行 SQL。

## 6. 数据、接口与运维规划

### 6.1 最小数据演进

| 数据域 | 建议增量 | 约束 |
|---|---|---|
| 会话 | 设备标识、刷新令牌元数据、撤销时间、最后使用时间。 | 服务端只保存不可逆或受控的令牌凭据；客户端令牌进入安全存储。 |
| 同步 | 队列状态、最近错误码、重试次数、下次重试时间、关联 ID。 | 错误信息脱敏；不保存 API Key、密码或完整认证头。 |
| 任务 | 草稿/已发布/已截止/已撤回/已归档状态、发布时间、截止时间、版本。 | 服务端时间为准；更新使用乐观版本或等效并发控制。 |
| 教学记录 | 任务关联、提交状态、统计快照或可复现聚合依据、导出审计。 | 保留期和清理规则可配置；只聚合必要字段。 |
| 运维 | 备份批次、恢复演练结果、服务健康摘要、安全审计事件。 | 仅记录脱敏元数据；备份内容与密钥分开受控存放。 |

### 6.2 服务端 API 原则

- 所有变更请求使用明确的认证、授权和输入校验，并返回可供客户端处理的稳定错误码，不能把内部堆栈、数据库结构或其他资源存在性暴露给调用者。
- 学习事件和离线任务操作携带稳定客户端操作 ID；服务端以“调用者 + 操作 ID”保证幂等，并返回已处理结果。
- 教师统计和 CSV 导出接口必须接收班级、任务及时间范围等显式过滤条件，并在服务端强制添加当前教师可见范围。
- 删除或清理采取软删除、保留期或明确恢复窗口中的一种可审计策略；任何不可恢复删除都需要二次确认和运维记录。
- 健康检查只暴露必要的非敏感状态；详细诊断仅对受控运维通道开放。

### 6.3 发布前运维清单

- 正式域名 DNS、证书申请/自动续期、HTTPS 跳转和 HSTS 已验证。
- 防火墙/安全组只开放 `80/tcp`、`443/tcp` 及受来源限制的 `22/tcp`；应用端口与数据库端口均不对公网开放。
- 部署使用独立非 root 账户、最小权限文件目录、受控环境变量和可回滚发布步骤。
- 已验证加密备份、保留期、恢复目标、恢复演练频率及负责人；恢复过程不覆盖生产数据。
- 访问日志、同步错误、认证异常、备份失败和证书续期失败具备脱敏告警与处理手册。
- 正式教学前由项目负责人确认学生数据合规要求、数据处理责任人、保留期限和应急联系人。

## 8. 风险与裁剪顺序

| 风险 | 应对 | 工期不足时的裁剪 |
|---|---|---|
| HTTPS、域名或证书准备滞后 | 第 1 周冻结域名与责任人；在独立测试域名提前演练证书续期。 | 不发布正式教学服务；不以 HTTP 替代。 |
| 持久令牌泄露或失效逻辑不一致 | 使用操作系统安全存储、短期访问令牌、可撤销刷新令牌和撤销测试。 | 不裁剪会话安全；裁剪“记住我”交互优化。 |
| 同步重试导致重复或丢失数据 | 稳定操作 ID、幂等处理、状态机、退避和离线夹具测试。 | 保留队列与手动重试；裁剪实时状态刷新。 |
| 任务状态和客户端缓存不一致 | 以服务端时间和状态机为唯一依据，客户端仅展示缓存。 | 保留发布/截止/撤回；裁剪草稿复制等便利功能。 |
| 导出造成学生数据过度暴露 | 服务端范围限制、字段白名单、导出审计和测试数据走查。 | 不裁剪授权和脱敏；裁剪复杂筛选与格式美化。 |
| 运维能力不足导致故障不可恢复 | 备份、恢复演练、告警和运行手册先于功能扩张。 | 不裁剪恢复；裁剪管理员桌面页。 |
| AI Provider 差异影响主流程 | 配置预检和安全降级，保持本地规则提示优先。 | 裁剪 Provider 预设数量，不裁剪 Key 本地化和 SQL 安全链路。 |
