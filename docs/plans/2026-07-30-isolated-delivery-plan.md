# 2026-07-10 至 2026-07-30 隔离式初步版交付计划

## 1. 目标

在 2026-07-30 前完成 SQLTeacher 初步可演示版。该计划优先保证 5 名成员的工作边界清晰、改动范围隔离、联调次数可控，避免多人同时修改同一批文件。

初步版必须跑通课堂演示所需的最小闭环：

```text
启动桌面程序
→ 初始化 SQLite 演示库
→ 查看表结构
→ 输入并执行 SQL
→ 展示结果或错误
→ 对高风险 SQL 给出拦截或确认
→ 调用 Ollama 生成 SQL 草案
→ 展示 AI 解释
→ 记录执行事件
→ 生成可运行 app-image
```

## 2. 隔离开发原则

- 7 月 10 日先冻结公共接口草案，后续只在固定联调窗口调整接口。
- 每个人负责一个主目录或主模块，日常开发不跨目录修改。
- UI、AI、数据库、测试打包通过 Application Service、DTO、测试桩和文档契约连接。
- 任何跨模块变更必须先写清楚影响范围，再由对应负责人确认。
- 每个成员每天只提交自己职责内的改动，避免“顺手修别人模块”。

## 3. 模块边界

| 成员 | 主边界 | 可修改范围 | 尽量不修改 |
|---|---|---|---|
| A | 架构与应用服务契约 | `src/main/java/com/sqlteacher/application`、公共 DTO、异常、配置、架构文档 | UI 页面、具体 JDBC 实现、Prompt 内容、打包脚本 |
| B | 数据库与 SQL 安全 | `src/main/java/com/sqlteacher/infrastructure/database`、SQL 执行、风险分析、demo 数据 | JavaFX 页面、AI Provider、打包脚本 |
| C | AI 与 NL2SQL | `src/main/java/com/sqlteacher/infrastructure/ai`、Prompt、AI DTO 校验、回归样例 | JDBC 实现、JavaFX 布局、打包脚本 |
| D | JavaFX 桌面端 | `src/main/java/com/sqlteacher/desktop`、`src/main/resources` 下 FXML/CSS | 数据库执行逻辑、AI 调用细节、安全规则 |
| E | 测试、打包与文档 | `src/test`、`docs`、打包脚本、验收材料 | 业务实现代码，除非修复测试夹具或构建问题 |

如当前代码目录尚未完全符合上述结构，先按现有结构就近落位，但不得扩大职责边界。

## 6. 每日执行规则

- 每天结束前提交可编译状态；若不能提交，记录阻塞原因。
- 每人每天优先完成自己模块内任务，不主动改他人模块。
- 每个任务完成时至少运行 `mvn test` 或对应的窄测试。
- 涉及 SQL 执行、安全规则、AI 输出格式的改动必须同步文档。
- 新增运行期文件必须加入 `.gitignore` 或放入明确的资源目录。
- UI 页面可以简陋，但不能阻塞主流程。

## 7. 接口变更规则

- 7 月 10 日后，公共接口默认冻结。
- 7 月 15 日、7 月 20 日、7 月 25 日是固定接口调整窗口。
- 非窗口期只允许修复编译错误、字段命名错误或严重阻塞问题。
- 接口变更必须写明影响成员、受影响文件、迁移方式和测试方式。
- 未经确认不得直接修改其他成员已经接入的 DTO、服务方法或数据库结构。

## 8. 初步版验收清单

7 月 30 日前必须全部满足：

- `mvn test` 通过。
- app-image 可生成并启动。
- SQLite demo 库自动初始化。
- SQL 练习页可以执行 `SELECT` 并展示结果。
- 错误 SQL 有可读错误提示。
- `DROP`、多语句等危险 SQL 被拦截。
- Ollama 状态可见。
- 自然语言至少能生成简单 `SELECT` 草案。

- 有 10 分钟演示脚本。
- 有已知限制清单。

## 9. 初步版不通过条件

出现以下任一情况，不能宣布 7 月 30 日初步版完成：

- 应用无法启动。
- SQLite 初始化失败。
- SQL 执行按钮不可用。
- 模型不可用时应用崩溃。
- AI 生成结果绕过风险检测直接执行。
- 禁止 SQL 可以被直接执行。
- 打包产物无法运行。
