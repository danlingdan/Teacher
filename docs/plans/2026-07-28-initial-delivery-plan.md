# 2026-07-28 初步版交付计划

## 1. 目标

在 2026-07-28 前完成 SQLTeacher 初步可演示版。该版本不追求完整首版范围，但必须跑通课堂演示所需的最小闭环：

```text
启动桌面程序
→ 初始化 SQLite 演示库
→ 查看表结构
→ 输入并执行 SQL
→ 展示结果或错误
→ 对高风险 SQL 给出拦截或确认
→ 调用 Ollama 生成 SQL 草案
→ 展示 AI 解释
→ 记录执行事件
→ 生成可运行 app-image
```

## 2. 交付边界

### 必须完成

| 编号 | 功能 | 说明 | 验收标准 |
|---|---|---|---|
| P0-01 | SQLite 演示库 | 固定 demo 数据、可一键初始化或恢复 | `student` 等示例表可查询 |
| P0-02 | SQL 执行服务 | 执行 SQLite 查询，返回列、行、错误信息 | `SELECT` 可展示表格结果 |
| P0-03 | SQL 风险检测 | 分类 `SELECT`、`INSERT`、`UPDATE`、`DELETE`、`DROP`、多语句 | 禁止语句默认拦截 |
| P0-04 | SQL 练习页 | SQL 输入、执行按钮、结果表格、错误区 | 低分辨率下主要控件可见 |
| P0-05 | Ollama 状态与调用 | 使用本地 `qwen3.5:0.8b` 或配置模型 | 服务可用时返回生成结果 |
| P0-06 | NL2SQL 草案 | 自然语言生成 SQL 草案，先允许简单 SELECT | 5 个典型查询样例通过 |
| P0-07 | AI 输出安全 | AI 结果必须进入风险检测，不直接执行 | 高风险 AI 结果不自动执行 |
| P0-08 | 事件记录 | 记录 SQL 执行、AI 生成、风险拦截 | `app.db` 中可查询记录 |
| P0-09 | 打包验证 | app-image 可运行 | 干净目录中启动成功 |
| P0-10 | 阶段文档 | 记录验证结果、已知限制、演示脚本 | 文档可指导 10 分钟演示 |

### 可以延后

| 功能 | 延后原因 | 最低替代 |
|---|---|---|
| MySQL 完整接入 | 7/28 前优先 SQLite 闭环 | 只保留接口和技术验证记录 |
| 知识库检索 | 非核心闭环 | 只保留接口或占位页面 |
| 学情看板图表 | 依赖事件数据积累 | 先提供事件表和简单统计 SQL |
| 完整设置页 | 业务逻辑优先 | 配置继续放 `application.properties` |
| 安装程序 MSI/EXE | app-image 已可演示 | 7/28 前只要求 app-image |

## 4. 每日执行规则

- 每天结束前必须提交可编译状态；若不能提交，记录阻塞原因。
- 每个任务完成时至少运行 `mvn test` 或对应的窄测试。
- 涉及 SQL 执行、安全规则、AI 输出格式的改动必须同步文档。
- 新增运行期文件必须加入 `.gitignore` 或放入明确的资源目录。
- UI 页面可以简陋，但不能阻塞主流程。

## 5. 初步版验收清单

7/28 前必须全部满足：

- `mvn test` 通过。
- app-image 可生成并启动。
- SQLite demo 库自动初始化。
- SQL 练习页可以执行 `SELECT` 并展示结果。
- 错误 SQL 有可读错误提示。
- `DROP`、多语句等危险 SQL 被拦截。
- Ollama 状态可见。
- 自然语言至少能生成简单 `SELECT` 草案。
- AI 生成 SQL 不会直接执行。
- 执行事件写入 `app.db`。
- 有 10 分钟演示脚本。
- 有已知限制清单。

## 6. 初步版不通过条件

出现以下任一情况，不能宣布 7/28 初步版完成：

- 应用无法启动。
- SQLite 初始化失败。
- SQL 执行按钮不可用。
- 模型不可用时应用崩溃。
- AI 生成结果绕过风险检测直接执行。
- 禁止 SQL 可以被直接执行。
- 打包产物无法运行。
