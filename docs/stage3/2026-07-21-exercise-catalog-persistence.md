# v0.3.0 题库持久化、导入导出与内置题库

## 1. 交付范围

- `JdbcExerciseManagementService` 实现题目列表、详情、创建、编辑、复制、启停和数据集读取。
- 题目编辑使用 `expectedVersion` 乐观版本检查；每次成功修改或启停后版本递增。
- 复制题目生成新 ID、版本重置为 1，并默认停用，避免未经检查的副本直接进入学生题库。
- 版本 1 JSON 题包同时包含题目及其依赖数据集；未知字段、错误版本、重复 ID 和无效领域数据会被拒绝。
- 整包导入使用单个事务。数据集内容相同可复用；内容冲突、题目冲突或依赖缺失时全部回滚。
- 应用数据库初始化时幂等写入 1 个学校数据集和 20 道启用题目。

## 2. 内置题目覆盖

| 分类 | 数量 | 主要知识点 |
|---|---:|---|
| 基础查询 | 4 | 选择列、排序 |
| 条件筛选 | 4 | 比较、文本条件、区间 |
| 聚合 | 4 | `COUNT`、`AVG`、`GROUP BY`、`HAVING` |
| 连接 | 4 | 内连接、左连接、连接聚合 |
| 子查询 | 4 | 标量、`IN`、`NOT EXISTS`、相关子查询 |

数据集初始化 SQL 只会在后续隔离练习数据库内执行，不会在 `app.db` 或用户当前连接中执行。

## 3. 安全与兼容边界

- 所有 JDBC 值均通过预编译参数写入。
- 导入失败会回滚已插入的数据集和题目。
- JSON 不接受未来格式版本，避免静默丢字段。
- 导出空 ID 列表表示导出全部题目；指定 ID 时任一题目不存在即失败。
- 初始化使用 `insert or ignore`，不会覆盖教师对内置题目的后续修改。
- 题目详情中的参考 SQL 和评测规则仍只从教师管理契约读取；学生契约不会暴露这些字段。
