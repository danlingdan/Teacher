# v0.3.0 题库与练习领域契约及 schema

## 1. 交付范围

本功能冻结 `v0.3.0` 题库、练习和确定性评测的第一版领域边界，为后续基础设施和 JavaFX 页面提供稳定契约。

已新增：

- 题目难度、数据集、题目定义和评测规则领域模型。
- 题目摘要与学生可见详情；学生 DTO 不暴露参考 SQL、数据集初始化 SQL 或评测规则。
- 教师题库管理契约：列表、读取、保存、复制、启停、版本化导入导出。
- 学生练习契约：开始会话、运行、提交和分级提示。
- 确定性评测结果：列、行、顺序、行数和 SQL 结构等分项结果。
- 练习运行/提交尝试结果与状态。

## 2. 数据安全边界

- `ExerciseView` 只包含题目说明、知识点、难度和版本。
- 参考 SQL、数据集初始化 SQL、评测规则只通过教师管理和评测服务使用。
- 学生运行 SQL 不直接访问 `app.db`，后续评测器必须创建隔离练习数据库。
- 分级提示最多三级，领域模型不提供标准答案字段。
- 题目修改使用 `expectedVersion` 契约，为基础设施乐观并发控制预留边界。

## 3. app.db schema 版本 3

新增表：

| 表 | 用途 |
|---|---|
| `exercise_datasets` | 保存版本化隔离数据集初始化 SQL |
| `exercises` | 保存题目、参考 SQL、评测规则、提示、版本和启用状态 |
| `exercise_sessions` | 保存一次学生练习会话、题目版本和提示使用次数 |
| `exercise_attempts` | 保存运行/提交 SQL、执行结果、评测反馈、耗时和错误分类 |

新增索引：

- `exercises_enabled_order`
- `exercise_sessions_exercise_started`
- `exercise_attempts_session_created`

schema 迁移保持顺序执行、单事务提交和失败回滚。旧版本 1/2 数据库会按原有迁移机制升级到版本 3，不删除连接配置和学习事件。
