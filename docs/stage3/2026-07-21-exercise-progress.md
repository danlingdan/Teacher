# v0.3.0 基础练习进度

## 1. 交付范围

新增 `ExerciseProgressService` 及 JDBC 实现，并在顶部导航提供“学习进度”页面。

概览展示：

- 练习会话数。
- 全部运行与提交尝试数。
- 提交次数与通过提交数。
- 提交通过率。
- 平均提交耗时。
- 累计使用提示次数。
- 至少通过一次的题目数。

逐题表格展示题目、知识点、全部尝试数、失败提交数、是否通过和最后尝试时间。未练习题目也会显示，便于识别尚未开始的内容。

## 2. 固定统计口径

- “尝试”包括 `RUN`、`PASSED` 和 `FAILED`。
- “提交”仅包括 `PASSED` 与 `FAILED`，不把仅运行查询计入通过率分母。
- “提交通过率”=`PASSED` 提交数 / (`PASSED` + `FAILED`) 提交数；无提交时为 0%。
- “平均提交耗时”只统计 `PASSED` 与 `FAILED` 记录的 `duration_ms`。
- “已完成题目”按出现过 `PASSED` 的不同 `exercise_id` 去重。
- 逐题“失败提交”只统计 `FAILED`，语法失败提交同样属于失败。

该口径在 application DTO 中集中表达，桌面层只负责格式化，避免 UI 自行重复计算。

## 3. 版本边界

本页是 v0.3.0 的个人练习进度摘要。按原扩展计划，时间趋势、常见错误排行、知识点薄弱项聚合、筛选和 CSV 导出仍属于 v0.4.0 基础学情看板，不在本次提前实现。

## 4. 验证策略

已编写固定数据夹具测试，覆盖运行不进入提交分母、通过率、平均耗时、提示数、完成题目去重和逐题失败次数。FXML 资源测试也包含新页面。

按用户指令暂未运行测试，只执行 `mvn -q -DskipTests compile`。本功能提交后，v0.3.0 功能范围即完成，下一步统一运行全量测试并修复问题。
