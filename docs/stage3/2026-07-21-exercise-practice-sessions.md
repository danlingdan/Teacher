# v0.3.0 学生练习会话、隔离数据集与尝试记录

## 1. 交付范围

- 学生题库查询仅返回启用题目和 `ExerciseView`，不会暴露参考 SQL、初始化 SQL 或评测规则。
- 开始练习时，在 `app-data/exercise-sessions/` 下创建会话专属 SQLite 数据库并执行版本化数据集脚本。
- 支持重置数据集、主动关闭会话和答对后自动关闭；关闭后删除临时数据库及可能的 WAL/SHM 文件。
- 会话 ID 必须是应用生成的 UUID，临时数据库路径经过归一化和父目录检查。
- 运行与提交只接受一条 `SELECT`，连接启用 SQLite `query_only`，查询超时 10 秒，结果最多返回 500 行。
- 每次运行/提交将 SQL、执行成功状态、结果列/行、反馈、耗时、错误码和时间写入 `exercise_attempts`。
- 会话记录题目版本；若教师在会话期间修改题目，旧会话会被判定为过期，要求重新开始。

## 2. 数据集脚本边界

数据集脚本来自教师题包，只在新建的隔离数据库执行。脚本拆分器识别字符串、行注释和块注释中的分号，避免错误拆分。初始化在事务内完成；失败时回滚并删除临时文件。

学生 SQL 始终经过 Java 侧风险分析；`INSERT`、`UPDATE`、`DELETE`、DDL、多语句和未知语句不会送入 JDBC 执行。

## 3. 会话生命周期

```text
开始 -> 创建隔离库 -> 运行/提示/重置 -> 提交
                                      -> 未通过：保持会话
                                      -> 通过：完成会话并清理隔离库
         \-> 主动关闭：完成会话并清理隔离库
```

提示使用次数写入会话，最多使用题目定义中的三级确定性提示。AI 不参与会话底座，Ollama 不可用不影响练习运行。

## 4. 验证策略

已编写测试覆盖只读运行、危险 SQL 拒绝、尝试持久化、数据集重置、提示级别、通过后关闭和临时文件清理。

按用户指定的 `v0.3.0` 节奏，本功能不单独运行测试，只执行 `mvn -q -DskipTests compile`；全量测试将在本版本全部功能完成后统一运行。

## 5. 下一功能

实现确定性评测器：执行参考 SQL 与学生 SQL，比较列、行、值、顺序、行数和必需 SQL 结构，并生成不泄露答案的分项反馈。
