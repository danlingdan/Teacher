# v0.3.0 确定性 SQL 评测与分级反馈

## 1. 判分原则

评测器不比较学生 SQL 与参考 SQL 的文本。两条查询在同一份临时数据集上独立执行，再按题目规则比较结果，因此别名、连接顺序、条件表达方式等写法不同但输出语义相同的查询可以通过。

支持的确定性规则：

- 结果列名称、数量和顺序。
- 结果行和值；无顺序要求时按多重集合比较，保留重复行语义。
- 结果行顺序。
- 指定结果行数。
- 必须出现的 SQL 结构关键字或短语。

JDBC 数值统一为规范化 `BigDecimal` 后比较，避免 `Integer`、`Long`、小数尾零等驱动表示差异造成误判；二进制值统一为 Base64；`NULL` 作为正常值参与比较。

## 2. 安全与隔离

- 学生 SQL 先经过 SQLite 方言风险分析，只允许单条 `SELECT`。
- 每次评测在 `app-data/exercise-evaluation/` 创建独立临时 SQLite 数据库。
- 初始化后启用 `pragma query_only`，参考查询和学生查询均不可修改数据集。
- 单条查询超时 10 秒，最多比较 5,000 行；超限返回明确错误而非截断后误判。
- 评测结束尽力删除数据库及 WAL/SHM 文件，清理失败只写日志，不向学生暴露本地路径。
- 反馈仅说明列、行、顺序、行数、结构或执行维度，不返回参考 SQL、参考结果或具体正确值。

## 3. 降级行为

评测完全由 Java 和 SQLite 完成，不依赖 Ollama。语法错误、安全拒绝、结果超限和临时环境失败都有确定性错误码与学生可理解反馈。

## 4. 验证策略

已编写测试覆盖：

- 不同写法但结果等价时通过。
- 内容相同但顺序错误时只暴露维度反馈。
- 必需 SQL 结构检查。
- 写操作安全拒绝。
- `NULL` 结果处理。
- 20 道内置题的参考 SQL 全部能够通过自身规则。

按当前约定不运行增量测试，仅执行 `mvn -q -DskipTests compile`。所有 v0.3.0 功能完成后统一运行全量测试。

## 5. 下一功能

实现教师题库管理和学生练习 JavaFX 页面，所有 JDBC 与评测调用通过后台执行器运行，并呈现加载、成功、失败和确认状态。
