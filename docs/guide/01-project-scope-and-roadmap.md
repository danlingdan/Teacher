# 项目范围与里程碑

## 1. 项目目标

SQLTeacher 是面向数据库课程教学、实验练习和课后辅导的桌面软件。首版目标是在 16 周内完成一个可演示、可安装、可测试的 Java 桌面版 MVP，并为后续服务器版扩展保留接口。

首版坚持“小闭环优先”：先完成 SQLite + SQL 执行 + AI 生成 + 风险检测 + 结果展示 + 安装包，再扩展 MySQL、知识库和学情统计。

当前短期目标：在 2026-07-30 前完成“初步可演示版”。详细拆分见 [2026-07-10 至 2026-07-30 隔离式初步版交付计划](../plans/2026-07-30-isolated-delivery-plan.md)。

## 2. 首版重点功能

| 优先级 | 功能 | 说明 | 验收方式 |
|---|---|---|---|
| P0 | 内置 SQLite 演示数据库 | 提供固定示例库、示例表和初始化脚本 | 首次启动后可直接查询 |
| P0 | SQL 编辑、预览、执行和结果展示 | 支持查询、错误展示、结果表格 | 能完成至少 10 个示例 SQL |
| P0 | 自然语言生成 SQL | 本地 Ollama 生成结构化输出，再由 Java 侧构造 SQL | 典型样例可稳定生成 |
| P0 | SQL 风险检测和教学提示 | 拦截危险 SQL，提示原因 | 禁止语句 100% 拦截 |
| P0 | Windows 自包含安装包 | 用户无需额外安装 JRE | 干净 Windows 环境可安装运行 |
| P1 | 外部 MySQL/MariaDB 数据库连接 | 支持连接测试、元数据读取、查询执行 | 合法配置可连接 |
| P1 | 本地 Ollama 模型调用 | 模型健康检查、超时、降级 | Ollama 不可用时有提示 |
| P1 | 基础课程知识检索 | 导入课程文档，支持关键词检索 | 可返回相关教学片段 |
| P1 | 基础学情统计 | 记录执行行为、成功率、常见错误 | 看板可展示统计 |
| P2 | UI 美化和体验优化 | 低分辨率适配、样式统一 | 主要流程不遮挡 |
| P3 | 服务器版接口预留 | 保留模块边界，不实现完整云端 | 架构不阻塞后续扩展 |

## 3. 首版暂不实现

- Oracle、SQL Server 等全部数据库适配。
- 企业级多租户权限。
- 完整云平台。
- 自动批改全流程。
- 复杂向量检索平台。
- 完全自治 AI Agent。
- 让模型直接执行 SQL。

## 4. 16 周实施计划

说明：阶段 0 和阶段 1 已有独立记录，后续执行以 2026-07-10 至 2026-07-30 隔离式初步版交付计划为短期优先级基准。若短期计划与本 16 周计划冲突，先保证 7 月 30 日初步版的 P0 链路可演示。

### 阶段 0：需求冻结与技术验证，第 1-2 周

目标：确认 MVP 范围，验证关键技术可行。

| 成员 | 任务 | 交付物 | 完成标准 |
|---|---|---|---|
| A | 建立 Maven 工程，完成架构草案，确定 Java 版本 | 工程骨架、架构图 | 项目可编译 |
| B | 验证 SQLite JDBC、MySQL JDBC、元数据读取 | 技术 Demo | 能读取表和列 |
| C | 验证 Ollama 调用、结构化 JSON 输出 | AI Demo | 能返回可解析 JSON |
| D | 验证 JavaFX 主窗口、FXML、CSS、基础页面切换 | UI Demo | 主窗口可启动 |
| E | 验证 jpackage 打包，制定测试计划初稿 | 打包 Demo、测试计划 | 能生成测试安装包 |

阶段验收：

- JavaFX 程序可启动。
- 能执行 SQLite 查询。
- 能连接 MySQL 或明确记录连接环境限制。
- 能调用 Ollama 或明确记录本地模型依赖。
- 能生成测试安装包。

### 阶段 1：基础工程与桌面骨架，第 3-4 周

目标：完成可启动的桌面应用骨架。

| 成员 | 任务 | 交付物 |
|---|---|---|
| A | 完成 Spring DI、配置、日志、异常处理、应用服务基础接口 | 应用启动骨架、配置模块 |
| B | 完成 SQLite 应用库初始化、数据库配置模型 | SQLite 初始化接口 |
| C | 完成 AI 配置模型、Ollama 状态检查接口 | AI 健康检查服务 |
| D | 完成主窗口、导航、设置页、连接页基础界面 | JavaFX 基础页面 |
| E | 建立单元测试模板、测试数据目录、缺陷记录模板 | 测试基础设施 |

阶段验收：

- 应用可启动并进入主窗口。
- 设置页可以展示数据库和模型配置。
- 日志文件正常输出。
- 至少有 1 个可运行测试。

### 阶段 2：SQLite 演示库与 SQL 执行引擎，第 5-6 周

目标：完成 SQLite 练习闭环。

| 成员 | 任务 | 交付物 |
|---|---|---|
| A | 编排 SQL 执行用例服务 | `SqlExecutionService` 草案 |
| B | 完成 demo.db、一键恢复、SQL 执行、事务、超时、行数限制、风险分类 | SQLite Adapter、风险分析器 |
| C | 提供错误解释接口占位，准备 AI 样例 | 错误解释 DTO |
| D | 完成 SQL 编辑器、执行按钮、结果表格、错误展示 | SQL 练习页 |
| E | 编写 SQLite 执行测试、异常场景测试 | SQLite 测试报告 |

阶段验收：

- 能打开内置库并查看表结构。
- 能执行 `SELECT` 并展示结果。
- 错误 SQL 能给出可读提示。
- 高风险 SQL 不会被直接执行。

### 阶段 3：NL2SQL 与教学机制，第 7-9 周

目标：实现自然语言到 SQL，并加入教学提示机制。

| 成员 | 任务 | 交付物 |
|---|---|---|
| A | 定义 NL2SQL 应用服务，整合 AI、SQL Builder 和执行安全 | `Nl2SqlService` |
| B | 实现 SQL Builder 对接、执行前安全校验 | SQL Builder、风险校验 |
| C | 完成 Prompt、结构化 DTO、JSON 校验、重试、SQL 解释、提示等级 | Prompt 模板、AI DTO |
| D | 完成自然语言输入区、生成 SQL 预览、提示面板、风险确认弹窗 | AI 助手页 |
| E | 建立至少 60 条 AI 回归测试样例 | AI 回归样例集 |

阶段验收：

- 用户输入自然语言后能生成 SQL 预览。
- 模型输出不可解析时不会崩溃。
- 高风险 SQL 必须二次确认。
- 至少 60 条回归样例有记录。

| D | 完成外部数据库连接配置页、表结构浏览界面 | 连接配置页 |

阶段验收：

- 能读取 MySQL 表结构。

## 5. 功能优先级裁剪规则

如果时间不足，按以下顺序保留：

| 优先级 | 必须保留 | 可以裁剪 |
|---|---|---|
| P0 | SQLite、SQL 执行、NL2SQL、安全检查、结果展示、安装包 | 无 |
| P1 | MySQL、学情统计、知识检索、教师导出 | 可降低界面复杂度 |
| P2 | UI 美化、Lucene 原型、服务器版预留接口 | 可只保留基础样式和接口草案 |
| P3 | 多数据库扩展、复杂图表、自动批改 | 首版不做 |

裁剪任何 P0 功能必须由项目负责人确认，并记录原因和替代方案。
