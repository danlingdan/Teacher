# 交付物与风险管理

## 1. 文档交付清单

项目结束时至少交付：

- 需求说明书。
- 架构设计文档。
- 数据库设计文档。
- 接口说明文档。
- 代码规范文档。
- 测试计划。
- 测试报告。
- 用户手册。
- 教师手册。
- 部署和安装说明。
- 演示脚本。
- 版本说明。
- 风险与问题清单。

- 测试报告。
- 用户手册。
- 教师手册。
- 答辩材料。
- 演示视频或演示脚本。
- 项目总结。

## 3. 风险管理

| 风险 | 影响 | 负责人 | 应对措施 |
|---|---|---|---|
| JavaFX 进度慢 | UI 延迟 | D | 先完成可用界面，后优化样式 |
| AI 输出不稳定 | SQL 生成失败 | C | 结构化输出、重试、回归测试 |
| JDBC 元数据差异 | 多库适配困难 | B | 每种数据库独立 Adapter |
| SQL 安全漏检 | 数据风险 | B、A | 规则拦截、权限限制、二次确认 |
| jpackage 打包失败 | 无法交付 | E | 第 2 周开始验证打包 |
| 公共接口频繁变更 | 联调困难 | A | 每周接口冻结，变更需通知 |
| 成员进度不均 | 阶段延期 | A | 每周拆小任务，及时调整 |
| 模型运行慢 | 课堂体验差 | C | 设置超时、低配模型、降级模式 |
| 真实库误操作 | 数据损失 | B | 默认限制、确认、审计、只读账号建议 |

## 4. 降级策略

当外部依赖不可用时，软件仍应尽可能保留核心教学能力。

| 依赖失败 | 降级行为 |
|---|---|
| Ollama 不可用 | 关闭自然语言生成，保留手写 SQL 和教学提示 |
| MySQL 不可用 | 保留 SQLite 演示库 |
| 知识库检索失败 | 保留 SQL 执行和 AI 解释 |
| 学情写入失败 | 提示记录失败，但不阻塞 SQL 查询结果展示 |
| 打包脚本失败 | 提供可运行 jar 和修复记录 |

## 5. 演示脚本建议

最终演示建议控制在 10-15 分钟：

1. 启动 SQLTeacher。
2. 展示首页和模型状态。
3. 打开 SQLite 演示库。
4. 查看表结构。
5. 手写一条 `SELECT` 并执行。
6. 输入自然语言生成 SQL。
7. 展示 SQL 预览和教学解释。
8. 执行并展示结果。
9. 尝试高风险 SQL，展示拦截或确认。
10. 打开知识库检索。
11. 打开学情看板。
12. 展示安装包或安装后启动效果。

## 6. 缺陷分级

| 级别 | 定义 | 处理要求 |
|---|---|---|
| P0 | 核心链路不可用、数据破坏、安全拦截失效 | 立即修复，阻塞发布 |
| P1 | 主要功能异常但有绕行方案 | 发布前修复 |
| P2 | 次要功能异常或体验问题 | 尽量修复，可记录已知问题 |
| P3 | 文案、样式、轻微优化 | 有时间再处理 |

## 7. 发布前清单

发布前必须确认：

- P0 功能全部可用。
- 禁止 SQL 语句 100% 拦截。
- 高风险 SQL 有确认。
- Windows 安装包可安装运行。
- 用户手册和教师手册可读。
- 测试报告已更新。
- 已知问题已记录。
- 演示脚本已跑通。

结论：5 人团队可以按模块并行开发，但必须通过公共接口、代码评审、测试和阶段演示控制集成风险。首版应坚持小步交付，先完成 SQLite + MySQL + Ollama + JavaFX 桌面闭环，再扩展知识库、学情和服务器版能力。
