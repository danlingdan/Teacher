# SQLTeacher v1.1 用户手册

## 云端教学与同步

应用启动后必须选择身份。已有账号可直接登录；注册时填写邮箱、昵称和至少 12 位密码，新账号默认获得学生身份；也可以选择“访客进入”体验本地功能。访客数据不会上传到账号，退出程序后也不能在其他设备恢复。

登录成功后，应用会先执行一次账号数据同步，再进入主界面。进入“云端教学”后，页面会自动选中第一个班级。选择班级即可查看教师发布的任务，点击“立即同步”可再次上传本机学习事件、下载其他设备的新记录。网络中断不会删除本地记录，下次可重新同步。

页面顶部状态条会显示加载、成功或错误信息。网络请求进行期间相关操作会暂时禁用，避免重复提交。

## 身份与功能权限

| 身份 | 可用功能 |
|---|---|
| 访客 | 首页、自由 SQL、闯关练习、AI 助手、表结构；仅本机使用，不显示云端和管理模块。 |
| 学生 | 访客功能，加课程知识、云端班级、任务查看和学习记录同步。 |
| 教师 | 学生功能，加教师题库、教师看板、数据库设置、班级成员管理和任务发布。 |
| 管理员 | 全部功能，并可查看所有云端班级。 |

右上角会显示当前昵称和身份。点击“切换身份”会撤销当前云端令牌并返回登录页。

需要使用网络 AI 时，填写厂商提供的 HTTPS Chat Completions 地址、模型名和 API Key。密钥只在当前应用进程内存中使用，退出应用后需要重新填写；不得在共享电脑展示密钥。

## 开始使用

安装后从桌面或开始菜单打开 SQLTeacher。首页顶部可进入自由 SQL、闯关练习、课程知识、AI 助手、表结构和设置。`Ctrl+1` 至 `Ctrl+9` 可依次切换九个顶部入口；窄屏下顶部导航可横向滚动。

## 自由 SQL 与安全确认

在“自由 SQL”选择连接、输入单条 SQL 后执行。查询默认限制结果数量；多语句、`DROP DATABASE`、`GRANT`、`REVOKE` 等操作会被阻止。高风险但允许的语句必须阅读风险说明并明确确认。不要在课程数据库使用高权限账号。

## 闯关练习

选择题目并开始会话，可运行草稿、使用分级提示并提交。最终判定由确定性结果规则完成，不由 AI 决定。每次练习使用隔离数据库，重置不会修改题库或应用数据库。

## AI 与课程知识

AI 助手仅生成草案，草案还需通过 Java 安全检查并复制到自由 SQL 页后由用户执行。Ollama 不可用时可继续手写 SQL。课程知识支持导入 UTF-8 `.txt`/`.md` 文档并展示来源；删除文档会同步删除索引。

## 备份与恢复

进入“设置与数据 → 版本与数据”：

1. “立即备份”保存 `app.db` 和演示库，最多保留最近 20 份。
2. 恢复前会校验 SQLite 完整性，并自动备份当前数据。
3. 恢复完成后应用会退出，重新打开即可使用恢复的数据。
4. “恢复内置演示库”只重建演示数据，不影响题库、学习记录和连接设置。

用户数据默认位于 `%LOCALAPPDATA%\SQLTeacher`，卸载应用不会删除该目录。
