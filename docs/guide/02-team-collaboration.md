# 团队分工与协作机制

## 1. 团队组织

团队共 5 人，采用“模块负责人 + 统一评审”的方式推进。每个人负责一个主方向，同时参与接口评审、代码评审和联调。

| 成员 | 角色 | 主要职责 |
|---|---|---|
| A | 项目负责人 / 后端架构 | 架构设计、工程搭建、核心服务、接口规范、进度协调 |
| B | 数据库与 SQL 执行负责人 | SQLite、MySQL、JDBC、元数据、SQL 执行、安全策略 |
| C | AI 与知识库负责人 | Ollama 调用、NL2SQL、结构化输出、知识检索、提示词管理 |
| D | JavaFX 前端负责人 | 桌面 UI、FXML/CSS、SQL 编辑器、结果表格、交互体验 |
| E | 测试、打包与文档负责人 | 测试用例、自动化测试、jpackage 打包、用户文档、演示材料 |

## 2. 成员详细任务

### A：项目负责人 / 后端架构

目标：保证系统结构清晰、接口稳定、模块可集成。

主要任务：

- 创建和维护 Maven 工程结构。
- 定义 `domain`、`application`、`infrastructure`、`desktop`、`server` 模块边界。
- 集成 Spring Boot 非 Web 容器或等价 DI 方案。
- 设计应用服务接口、DTO、异常类型和统一返回结构。
- 负责配置管理、日志管理和全局异常处理。
- 组织每周例会、接口评审和阶段验收。
- 审核核心模块代码。
- 维护架构设计文档和接口说明。

核心交付物：

- Maven 项目结构。
- 项目基础包结构。
- 应用服务接口。
- 全局异常处理。
- 配置与日志模块。
- 架构设计文档。
- 阶段版本发布记录。

### B：数据库与 SQL 执行负责人

目标：保证数据库连接、SQL 执行和安全控制可靠。

主要任务：

- 实现 `DatabaseAdapter` 接口。
- 实现 `SqliteDatabaseAdapter`。
- 实现 `MySqlDatabaseAdapter`。
- 设计数据库连接配置模型。
- 集成 SQLite JDBC。
- 集成 MySQL Connector/J 或 MariaDB Driver。
- 集成 HikariCP。
- 实现连接测试。
- 实现表、列、主键、外键、索引等元数据读取。
- 实现 SQL 执行结果标准化。
- 实现查询行数限制、超时控制和事务控制。
- 实现 SQL 分类、风险分析和执行拦截。
- 提供 `demo.db` 初始化和一键恢复能力。

核心交付物：

- `DatabaseAdapter` 及实现类。
- `demo.db` 示例数据库。
- 数据库连接配置页所需接口。
- SQL 执行引擎。
- SQL 风险分析模块。
- 数据库元数据标准化模型。
- 数据库相关测试用例。

### C：AI 与知识库负责人

目标：保证 AI 输出可控、可校验、可回归。

主要任务：

- 定义 `AiModelProvider` 接口。
- 使用 Spring AI、WebClient 或简单 HTTP 客户端调用 Ollama。
- 实现模型健康检查。
- 设计 NL2SQL Prompt 模板。
- 设计结构化 JSON 输出格式。
- 实现 Jackson 反序列化。
- 实现 Jakarta Validation 校验。
- 实现结构化输出失败重试。
- 与 B 协作完成 SQL Builder 接口。
- 实现 SQL 解释、错误解释和分步提示。
- 建立 AI 回归测试样例。
- 实现课程文档导入。
- 实现 SQLite FTS5 基础检索。
- 管理 Prompt 版本记录。

核心交付物：

- `AiModelProvider` 接口及 Ollama 实现。
- NL2SQL DTO。
- Prompt 模板。
- 模型配置界面所需接口。
- AI 回归测试集。
- 课程知识检索模块。
- SQL 解释和教学提示模块。

### D：JavaFX 前端负责人

目标：保证桌面端核心流程清晰、稳定、不卡顿。

主要任务：

- 搭建 JavaFX 主窗口。
- 设计首页、数据库连接页、SQL 练习页、AI 助手页、知识库页、学情看板页、设置页。
- 使用 FXML + CSS 分离界面与样式。
- 接入 Spring 管理 Controller 或统一 Controller 工厂。
- 实现 SQL 编辑器。
- 实现执行结果表格。
- 实现错误提示、风险确认弹窗和加载状态。
- 实现数据库表结构浏览。
- 实现 Ollama 状态显示。
- 实现学情统计图表。
- 与 A 协作统一前端调用 Application Service。
- 保证低分辨率屏幕下界面可用。

核心交付物：

- JavaFX 主程序。
- FXML 页面。
- CSS 样式。
- SQL 编辑器界面。
- 结果表格界面。
- 数据库连接界面。
- 学情看板。
- 设置页面。

### E：测试、打包与文档负责人

目标：保证软件可验证、可安装、可交付。

主要任务：

- 制定测试计划。
- 编写核心单元测试。
- 组织集成测试。
- 维护测试数据。
- 维护 AI 回归测试报告。
- 验证 Windows 安装包。
- 配置 jlink 和 jpackage。
- 在干净 Windows 环境测试安装、启动、卸载和升级。
- 编写用户手册。
- 编写教师使用说明。
- 编写部署说明。
- 整理答辩 PPT 或演示脚本。
- 维护缺陷列表和验收清单。

核心交付物：

- 测试计划。
- 测试用例。
- 测试报告。
- 打包脚本。
- Windows 安装包。
- 用户手册。
- 教师手册。
- 演示脚本。
- 项目最终交付清单。

## 3. 每周协作机制

每周至少两次同步：

| 时间 | 内容 | 必须产出 |
|---|---|---|
| 周一 | 明确本周目标、接口变更、风险点 | 本周任务清单 |
| 周五 | 演示本周成果、合并代码、更新缺陷列表 | 演示记录、问题清单 |

每次会议必须记录：

- 本周完成内容。
- 未完成原因。
- 下周任务。
- 阻塞问题。
- 需要其他成员配合的事项。
- 公共接口或数据库结构是否变化。

## 4. 每日沟通

建议每天用 10 分钟同步：

- 昨天完成什么。
- 今天准备做什么。
- 当前是否被阻塞。
- 是否修改了公共接口。
- 是否需要他人评审或联调。

## 5. 接口变更规则

公共接口包括：

- Application Service。
- DTO。
- `DatabaseAdapter`。
- `AiModelProvider`。
- 配置模型。
- 数据库表结构。
- 前端页面调用契约。
- AI 结构化输出格式。
- Prompt 输入输出约定。

任何公共接口变更必须：

1. 先在群内说明影响。
2. 更新接口文档或对应专题文档。
3. 通知相关负责人。
4. 给调用方预留同步修改时间。
5. 不直接破坏主分支。

## 6. 代码评审分工

| 改动类型 | 必须评审人 |
|---|---|
| 架构、公共接口、模块边界 | A |
| 数据库连接、SQL 执行、安全策略 | A、B |
| AI 调用、Prompt、结构化输出 | A、C |
| JavaFX 主流程、页面交互 | D，必要时 A |
| 测试、打包、安装脚本 | E，必要时 A |
| 文档和演示材料 | E 或对应模块负责人 |

## 7. 合并前检查清单

- 是否符合包结构和模块边界。
- 是否存在无意义大文件提交。
- 是否修改了公共接口。
- 是否更新了相关文档。
- 是否有必要测试。
- 是否处理异常。
- 是否记录敏感信息。
- 是否阻塞 UI 线程。
- 是否绕过 SQL 安全检查。
- 是否破坏已有功能。
- 是否能在本地启动或通过相关测试。
