# SQLTeacher v1.0 数据库配置说明

## SQLite

内置 `app` 连接保存题库、学习事件、知识索引与非敏感设置；内置 `demo` 连接用于自由 SQL 演示。新增 SQLite 连接时填写本机数据库绝对路径。应用不会在连接配置中保存密码。

## MySQL 与 MariaDB

在“设置与数据 → 数据库连接”新增连接，填写主机、端口、库名、用户名，并保持“只读连接”启用。密码仅保存在当前进程内存中，用于连接测试和本次会话，关闭应用后清除。

建议由管理员创建专用账号，只授予课程库的 `SELECT` 和元数据读取权限。不要使用 root 或拥有写权限的教学账号。客户端还会执行以下保护：单语句限制、查询超时、结果上限、JDBC 只读模式、方言风险分析和禁止语句拦截。客户端保护不能替代服务器权限。

## 故障定位

- 认证失败：检查账号、临时密码和服务端允许的主机。
- 网络失败：检查地址、端口、防火墙和数据库服务状态。
- 权限不足：让管理员授予目标库最小 `SELECT` 权限，不要扩大为全局权限。
- MySQL/Ollama 不可用：切换回内置 `demo`，SQLite 练习、评测和统计仍可使用。

日志与导出不得包含数据库密码。向他人提供诊断材料前，应检查主机名、用户名、课程文档和学生数据。
