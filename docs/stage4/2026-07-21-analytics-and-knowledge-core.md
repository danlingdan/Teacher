# v0.4 学情与知识检索核心服务

## 已实现

- `app.db` schema 升至版本 4，新增知识文档、知识切片与 SQLite FTS5 索引。
- 新增统一学情统计契约，集中定义时间、题目、知识点和错误类型筛选。
- 学情报告与 CSV 由同一服务生成，固定通过率、完成率、平均尝试次数和薄弱度口径。
- CSV 使用 UTF-8 BOM、CRLF，并记录筛选范围、生成时间与统计口径。
- 本地知识库支持 UTF-8 `.txt`、`.md`、`.markdown`，单文件限制 2 MiB。
- 文档导入采用事务、SHA-256 去重和最多 800 字符切片；删除同步清理 FTS5 索引。
- 学习事件新增练习尝试、通过、失败、提示和知识检索类型，不记录 SQL、查询原文或文档全文。
- 新增学习会话、尝试和事件的事务清理服务。

## 安全边界

- 课程材料只存储在本地 `app.db`，不发送到 Ollama 或外部服务。
- FTS 查询使用参数化 SQL，并将用户输入转为受控短语表达式。
- 知识内容不能改变 SQL 风险分析、练习评测或 AI 安全规则。
- 清理学习数据不删除题库、连接配置或知识文档。
