# SQLTeacher SQL 教学与智能练习软件 V1.0

## 用户操作说明书（初稿）

> 软件名称和版本号均为候选值。正式排版前必须与申请表确认一致，并替换为冻结版本的真实界面截图。

## 1. 软件概述

SQLTeacher 是面向数据库课程的桌面练习软件。软件将 SQLite 演示数据库、SQL 编辑执行、
结果展示、表结构浏览、风险分析和本地 AI 辅助整合在同一桌面界面中，帮助学生在可控环境下
完成基础 SQL 学习和实验练习。

软件的核心设计原则是“先分析、后执行”。用户提交的 SQL 由应用服务统一处理，风险分析模块
识别语句类型、多语句和禁止操作。AI 模型只生成草案和解释，不持有数据库连接，也不能直接执行 SQL。

## 2. 运行环境

- 操作系统：Windows 10 或 Windows 11 64 位。
- Java 环境：开发运行要求 Java 21；最终自包含 app-image 可携带运行时。
- 显示环境：支持常见桌面分辨率，建议不低于 1366×768。
- 本地数据：软件在 `app-data` 目录保存应用数据库、演示数据库和日志。
- AI 功能：如需使用 AI 辅助，应在本机启动 Ollama 并准备兼容模型。

## 3. 软件启动

开发环境可在项目目录运行：

```powershell
mvn javafx:run
```

打包版本应从生成的应用目录启动 SQLTeacher。首次启动时，软件准备本地应用数据库和 SQLite
演示数据库。初始化失败时，界面应显示可理解的错误信息，不要求用户阅读 Java 堆栈。

## 4. 主界面

主窗口顶部显示软件品牌和导航入口。当前主要入口包括“SQL 练习”和“表结构”。页面切换在主窗口
内部完成，不额外打开独立业务窗口。耗时操作期间，全局加载遮罩提示当前状态，避免重复提交。

## 5. SQL 练习

用户进入“SQL 练习”页面后，在编辑区输入 SQL。软件可以填充教学示例，便于快速体验查询流程。
点击执行后，软件先分析 SQL 的类型和风险，再决定允许执行、要求确认或直接拦截。

对于允许执行的查询，结果区域展示列名和数据行；当结果数量达到限制时，软件显示截断状态。
对于空结果，界面使用空状态提示；对于语法或数据库错误，界面显示面向学习者的错误信息。

典型示例：

```sql
SELECT name, score
FROM student
WHERE score >= 60
ORDER BY score DESC;
```

## 6. SQL 风险提示

软件按语句类型和内容给出风险等级。只读查询通常为低风险；数据修改和结构修改需要额外确认；
多语句及禁止操作默认拦截。风险提示用于帮助学生理解语句影响，不能替代真实数据库的权限和备份机制。

主要规则包括：

- 默认阻止一次执行多条 SQL。
- 默认阻止 `DROP DATABASE`、`GRANT` 和 `REVOKE` 等禁止操作。
- 对 `UPDATE`、`DELETE`、`ALTER` 和 `DROP` 等高风险操作显示确认信息。
- 查询结果设置最大行数，避免一次加载过多数据。

## 7. 表结构浏览

用户进入“表结构”页面后，可查看当前演示数据库的表和字段信息。表结构信息包括表名、列名、
字段类型、可空性和主键标记等。该信息也可作为 AI 生成 SQL 时的数据库上下文。

## 8. AI 辅助 SQL

当本地 Ollama 可用时，软件把自然语言需求和必要的表结构信息组织为提示，要求模型返回结构化 JSON。
Java 侧解析并校验返回内容，生成 SQL 草案和教学解释。草案必须展示给用户，并在执行前再次进入 SQL
风险分析流程。

当 Ollama 未启动、模型不可用、请求超时或输出无法解析时，软件返回降级提示，手写 SQL 功能仍可使用。

AI 输出可能不准确。用户应核对表名、列名、筛选条件和数据影响，不应把模型建议视为数据库权限控制。

## 9. 数据和日志

软件的本地运行数据位于 `app-data` 目录。`app.db` 保存应用事件等数据，`demo.db` 保存教学演示数据，
日志目录用于诊断启动和运行问题。配置、日志和数据库中不得保存明文密码或模型密钥。

## 10. 异常处理

- 数据库初始化失败：检查目录写权限和磁盘空间后重试。
- SQL 语法错误：根据错误提示检查关键字顺序、列名、括号和逗号。
- 查询没有结果：确认筛选条件和演示数据内容。
- Ollama 不可用：启动本地服务并确认模型已经安装，或继续使用手写 SQL。
- AI 请求超时：简化自然语言描述后重试。

## 11. 安全说明

软件不会允许 AI 模型直接访问 JDBC 连接。所有 SQL 执行均应通过应用服务和风险分析路径。
对真实或重要数据库进行修改前，用户仍应建立备份、配置最小权限账号并独立复核 SQL。
