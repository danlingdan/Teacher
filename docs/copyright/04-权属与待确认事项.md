# 权属与待确认事项

以下内容不能仅凭代码仓库可靠判断，是继续制作正式软著材料前的必答项。

## 1. 已确认的申请主体原则

- 软件按多人共同著作权准备。
- 学校作为第一著作权人/第一署名单位。
- 开发方式按合作开发准备，不填写“独立开发”。

## 2. 仍需补全的主体信息

- [ ] 学校在法人证书或其他有效证件上的法定全称是什么？
- [ ] 学校是否作为全体著作权人的申请代表？
- [ ] 学校是否允许与学生或其他自然人共同登记为著作权人？
- [ ] 学校办理软著的归口部门、盖章和成果审批流程是什么？
- [ ] 其余共同著作权人的真实姓名、证件类型、证件号码和署名顺序是什么？
- [ ] 所有自然人申请信息是否与身份证件完全一致？

## 3. 合作开发和权利归属

仓库提交记录显示至少存在多个提交身份，项目文档也描述了 5 人分工。现已确认按合作开发、多人共同
著作权准备，学校排第一。仍须用学校认可的权属材料确认最终权利人范围和排序。

- [ ] 是否有课程任务书、委托开发合同、劳动关系文件或合作开发协议？
- [ ] 五名成员是否全部列为共同著作权人，还是仅列权属协议确定的人员？
- [ ] 学校、指导教师、团队负责人或出资方是否对成果归属有书面要求？
- [ ] 如由学校代表申请，是否已有其他共同著作权人的书面同意、授权或权属约定？
- [ ] Git 提交邮箱和昵称能否对应到真实开发者身份？

在上述问题未确认前，不建议提交申请。

## 4. 版本和日期

- [ ] 最终登记名称是否采用“SQLTeacher SQL 教学与智能练习软件”？
- [ ] 软件简称是否为“SQLTeacher”？
- [ ] 登记版本是否为 `V1.0`？
- [ ] 哪个 Git 提交作为登记版本？当前候选为 `55428fc5`。
- [ ] 开发完成日期是哪一天？该日期不应早于实际登记版本完成时间。
- [ ] 软件是否已经向不特定公众提供、发布或销售？
- [ ] 如已发表，首次发表日期和地点是什么？

## 5. 申请范围和事实核对

- [ ] 申请功能范围是否只描述当前已经实现的 SQLite 教学闭环？
- [ ] 是否将本地 Ollama 作为可选运行依赖，而不是软件自研模型？
- [ ] 是否同意不把 JavaFX、Spring、Jackson、JDBC 驱动等第三方组件主张为自研源程序？
- [ ] 最终软件截图是否来自登记基线，而不是 mock、原型或后续未冻结版本？
- [ ] 软件界面、文档和源代码中是否包含需要脱敏的学生数据或个人信息？

## 6. 建议保存的内部证据

- 冻结提交的 Git 提交号、提交记录和完整源码归档。
- 项目任务书、分工记录、合作协议和权利归属文件。
- 构建日志、测试报告、演示记录和版本发布日期记录。
- 最终安装包或 app-image 的哈希值。
- 提交版申请表、源程序和说明书的最终 PDF 及文件哈希值。
