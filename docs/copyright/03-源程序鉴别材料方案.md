## 2. 建议纳入范围

按相对路径字典序连续拼接以下自研生产源程序：

```text
src/main/java/com/sqlteacher/application/**/*.java
src/main/java/com/sqlteacher/domain/**/*.java
src/main/java/com/sqlteacher/infrastructure/**/*.java
src/main/java/com/sqlteacher/desktop/**/*.java
src/main/java/com/sqlteacher/*.java
```

拼接清单必须固定，文件之间可加入一行文件路径注释，但不得为了凑页数重复或打乱代码。

## 3. 不纳入范围

- `src/test` 下的测试和 mock 测试夹具。
- Maven 下载的 JavaFX、Spring、Jackson、SQLite JDBC 等第三方源代码。
- `target`、`app-data`、日志、数据库和安装产物。
- `.idea`、本地配置、密钥、账号、密码和个人隐私数据。
- Git 元数据、评审记录和与软件运行无关的项目管理文档。

FXML 和 CSS 属于自研界面资源，但当前 Java 源程序已经超过 60 页所需规模。为保证程序材料边界清晰，
初稿方案只使用 Java 生产源代码；如办理机构要求体现界面资源，再单独调整取样清单。

## 4. 连续页取样

当前 Java 源程序约 5,111 行。按每页至少 50 行估算，整体超过 60 页，因此常规方案为：

1. 先按固定顺序生成完整连续源程序母版。
2. 每页排 50 至 55 行，保持代码顺序不变。
3. 从母版取最前连续 30 页。
4. 从母版取最后连续 30 页。
5. 检查前后两段内部无跳页、无漏行、无重复页。

“前 30 页”和“后 30 页”应相对于同一份完整母版确定，不能分别挑选看起来重要的文件。

## 5. 排版规则

- A4 纵向、单面、黑白。
- 页眉统一使用最终确认的软件全称和版本号。
- 右上角连续标注页码。
- 使用等宽字体，字号和行距应保证每页不少于 50 行且代码可辨认。
- 文件路径、包名、类名和代码内容不得截断或乱码。
- 删除真实密钥、密码和个人敏感信息；当前项目不应把这些内容纳入版本库。
- 申请表、说明书、源程序页眉中的软件名称、版本号和著作权人署名保持一致。

## 6. 例外交存和保密

如果源程序包含不宜公开的核心内容，可在正式办理前评估《计算机软件著作权登记办法》第十二条规定的
例外交存方式。是否采用例外交存属于申请策略和权属决策，本初稿不自动选择。
